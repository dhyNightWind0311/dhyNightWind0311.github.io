线段树合并

· 2022-11-22 · # 学习笔记

by TheBigYellowDuck

背景

封闭的第一次模拟赛遇到了这个科技，因为完全不会喜提 20pts20 \text{pts}20pts。

事后发现这个东西很好玩，赶紧学了一下。

前置知识

线段树合并是将两棵值域相同的权值线段树合并的算法，一般用来合并动态开点权值线段树。

普通权值线段树也可以合并，但是意义不大。

其本质是对应位置相加，虽然看起来很暴力，但一次合并的复杂度是优秀的 O(log⁡n)\mathcal{O}(\log{n})O(logn)。

实现上跟正常动态开点权值线段树几乎一致，差别只在合并上。

实现参考：

int merge(int u, int v, int l, int r) {
    if (u == 0 || v == 0)
        return u + v;
    if (l == r) {
        sum[u] += sum[v];
        return u;
    }
    int mid = (l + r) >> 1;
    ls[u] = merge(ls[u], ls[v], l, mid);
    rs[u] = merge(rs[u], rs[v], mid + 1, r);
    return pushup(u), u;
}

Problem 1

有 nnn 个数，第 iii 个数的颜色为 cic_ici​。初始时每个数所属集合只包含自己。进行 qqq 次操作：

  • 合并 aaa 和 bbb 所在集合。

  • 查询 xxx 所在集合中颜色为 yyy 的数的个数。

数据范围：ci≤n,q≤2×105c_i\leq n,q\leq 2\times10^5ci​≤n,q≤2×105。

Sol

线段树合并模板。

操作一容易想到并查集，操作二是权值线段树上单点查询问题。

于是用并查集维护集合，合并集合的同时把两个集合的权值线段树合并。在权值线段树上单点查询即可。注意并查集的合并方向要和权值线段树保持一致。

时间复杂度 O(nlog⁡n+qlog⁡n)\mathcal{O}(n\log{n}+q\log{n})O(nlogn+qlogn)。

题目来源： ABC183F Confluence

Problem 2

给定一张 nnn 个点 mmm 条边的无向图，每个点有权值 viv_ivi​。进行 qqq 次操作：

  • 在点 xxx 和点 yyy 之间连边。

  • 查询点 xxx 所在连通块内权值第 kkk 小的点的编号。不存在输出 −1-1−1。

数据范围： vi≤m≤n≤105v_i\leq m\leq n\leq 10^5vi​≤m≤n≤105，q≤3×105q\leq 3\times 10^5q≤3×105。

Sol

换汤不换药。

维护连通块还是在维护集合。查询可以在权值线段树上完成，直接无脑线段树合并。

和上一题的写法几乎一样，只是要查询的东西不同。

时间复杂度 O(nlog⁡n+qlog⁡n)\mathcal{O}(n\log{n}+q\log{n})O(nlogn+qlogn)。

题目来源： HNOI2012 永无乡

Problem 3

给定一棵根为 111 的树，节点 iii 的权值为 pip_ipi​。对于所有 1≤i≤n1\leq i\leq n1≤i≤n 求节点 iii 的子树中权值大于 pip_ipi​ 的节点个数。

数据范围： n≤105n\leq 10^5n≤105，pi≤109p_i\leq 10^9pi​≤109。

Sol

先离散化。

注意到节点 iii 的答案其实就是在权值线段树上区间 [pi+1,n][p_i+1,n][pi​+1,n] 的和。考虑在每一个叶子节点建权值线段树，不断向上合并，合并到节点 iii 时直接可以求出 iii 的答案。

时间复杂度 O(nlog⁡n)\mathcal{O}(n\log n)O(nlogn)。

题目来源：USACO2017 Promotion Counting

Problem 4

村落里一共有 nnn 座房屋，形成一个树。发放 mmm 次救济粮，每次选择两个房屋 x,yx, yx,y，然后对于 xxx 到 yyy 的路径上每座房子里发放一袋 zzz 类型的救济粮。

当所有的救济粮发放完毕后，求每座房子里存放的最多的救济粮类型。

数据范围：n,m,z≤105n,m,z\leq 10^5n,m,z≤105。

Sol

先通过树上差分把链上的操作转化为单点操作。

权值线段树记录出现次数的最大值及其类型，从叶子节点向上合并即可。

时间复杂度 O(nlog⁡n+mlog⁡n)\mathcal{O}(n\log n+m\log n)O(nlogn+mlogn)。

题目来源：P4556 雨天的尾巴

Problem 5

游戏的地图为一棵 nnn 个节点的树，现在有 mmm 个玩家，第 iii 个玩家的起点为 sis_isi​ 终点为 tit_iti​。所有玩家在第 000 秒同时从自己的起点出发，以每秒跑一条边的速度，不间断地沿着最短路径向着自己的终点跑去，跑到终点后该玩家退出游戏。

每个节点上都存在一个观察员。在节点 jjj 的观察员会在第 wjw_jwj​ 秒观察玩家，玩家能被观察员观察到当且仅当玩家在第 wjw_jwj​ 秒也正好到达了节点 jjj。求每个观察员会观察到的人数。

数据范围：n,m≤3×105n,m\leq 3\times 10^5n,m≤3×105。

Sol

考虑统计每条路径对观察员做出的贡献。

对于路径 (s,t)(s,t)(s,t)，设 l=lca⁡(s,t)l=\operatorname{lca}(s,t)l=lca(s,t)，把路径拆成上行路段 (s,l)(s,l)(s,l) 和下行路段 (l,t)(l,t)(l,t) 两段。不妨认为 lll 在上行路段上。

设 depudep_udepu​ 为节点 uuu 的深度，观察员位置为 kkk，分类讨论：

  • 若 kkk 在路径 (s,l)(s,l)(s,l) 上，则想要做出贡献需要满足

deps=depk+wkdep_s=dep_k+w_k deps​=depk​+wk​

  • 若 kkk 在路径 (l,t)(l,t)(l,t) 上，则想要做出贡献需要满足

deps−depl+depk−depl=wkdep_s-dep_l+dep_k-dep_l=w_k deps​−depl​+depk​−depl​=wk​

depk−wk=2×depl−depsdep_k-w_k=2\times dep_l-dep_s depk​−wk​=2×depl​−deps​

考虑在每个点上开一棵权值线段树。对于第一个式子，我们可以在路径 (s,l)(s,l)(s,l) 上的每一个点的权值线段树上标记 depsdep_sdeps​，则对于观察员 kkk 只需要查询权值线段树上单点 depk+wkdep_k+w_kdepk​+wk​ 的值，就可以知道经过 kkk 的上行路段数量。

进一步考虑，在一条路径上全部标记 depsdep_sdeps​ 可以通过树上差分转化为在 sss 的权值线段树上标记 depsdep_sdeps​ 而在 falfa_lfal​ 的权值线段树上撤销，从下往上进行线段树合并，就可以得到答案。

对于第二个式子同理。在 ttt 的权值线段树上标记 2×depl−deps2\times dep_l-dep_s2×depl​−deps​ 并在 lll 的权值线段树上撤销，从下往上合并答案，对于观察员 kkk 只需要查询权值线段树上单点 depk−wkdep_k-w_kdepk​−wk​ 的值，就可以知道经过 kkk 的下行路段数量。

注意这里 falfa_lfal​ 和 lll 的区别是因为我们把 lll 归入了上行路段，其实反过来也是一样的。

Problem 6

给定一颗有 nnn 个叶节点的二叉树。每个叶节点都有一个权值 pip_ipi​，所有叶节点的权值构成一个 1∼n1 \sim n1∼n 的排列。

对于这棵二叉树的任何一个结点，保证其要么是叶节点，要么左右两个孩子都存在。
现在你可以任选一些节点，交换这些节点的左右子树。

在最终的树上，按照先序遍历遍历整棵树并依次写下遇到的叶结点的权值构成一个长度为 nnn 的排列，你需要最小化这个排列的逆序对数。

数据范围：n≤2×105n\leq 2\times 10^5n≤2×105。

Sol

考虑一个节点的子树的先序遍历序列中逆序对的来源有三种：

  • 左子树内部的逆序对

  • 右子树内部的逆序对

  • 横跨左右子树产生的逆序对。

注意到交换子树的操作只会影响第三种逆序对的数量，考虑对交换和不交换分别计数。

设当前要合并的左右子树的根分别为 uuu 和 vvv。若不交换左右子树，产生的逆序对数量为

∑rs(u)×∑ls(v)\sum rs(u)\times \sum ls(v) ∑rs(u)×∑ls(v)

若交换左右子树，产生的逆序对数量为

∑ls(u)×∑rs(v)\sum ls(u)\times \sum rs(v) ∑ls(u)×∑rs(v)

递归的时候对每一个节点的两式取最小值即可。

读入是一个递归的形式，所以可以一边读入一边做，正好也满足从叶子向上合并的顺序。

时间复杂度 O(nlog⁡n)\mathcal{O}(n\log n)O(nlogn)。

题目来源：P3521 POI2011 Tree Rotations

Problem 9

给定 nnn 个点的有根树，每个点权值为 wiw_iwi​。对于每个点 iii， 求以 iii 为根的子树中权值集合的 mex\text{mex}mex，即集合中未出现的最小正整数。

数据范围：n,wi≤105n,w_i\leq 10^5n,wi​≤105。

Sol

只要知道如何用权值线段树处理 mex\text{mex}mex，就可以用线段树合并。

设当前递归到的权值线段树的根为 uuu，值域区间为 [l,r][l,r][l,r]，考虑如下情况：

  • 若 u=0u=0u=0，说明已经递归到了边界，答案为 lll。

  • 若子树和等于值域区间长度，说明子树已经被填满，答案为 r+1r+1r+1。

  • 若左子树的答案小等于于 ⌊l+r2⌋\left\lfloor\dfrac{l+r}{2}\right\rfloor⌊2l+r​⌋，说明左子树还未填满，答案为左子树的答案。

  • 否则，左子树填满，答案为右子树的答案。

int query(int rt, int l, int r) {
    if (!rt) return l;
    if (sum[rt] == r - l + 1) return r + 1;
    int mid = (l + r) >> 1;
    int res = query(ls[rt], l, mid);
    if (res <= mid) return res;
    return query(rs[rt], mid + 1, r);
}

用处理树形问题的常规手段，从叶子向上进行线段树合并即可。

时间复杂度 O(nlog⁡n)\mathcal{O}(n\log n)O(nlogn)。
